# CLAUDE CODE PROMPT: PROJECT START

**Role:** Expert Game Developer & Software Architect.
**Project Name:** "Resilience Light" (Світло Стійкості).
**Genre:** Economic strategy, crisis management simulator, state survival.
**Platform:** Web Browser (HTML5 Canvas + LocalStorage, No Backend).
**Visual Style:** Military-tech, "Digital Alert Map", Dark Mode, Glassmorphism, neon accents.

## 1. TECH STACK & ARCHITECTURE
* **Map/Graphics Engine:** Pure HTML5 Canvas API or PixiJS (WebGL). **Strict Rule: No heavy DOM frameworks (Vue/React) for rendering the game map and active objects.** UI overlays can be basic HTML/CSS over the canvas.
* **Interaction:** "Hit Areas" (polygons) overlaying the Canvas to handle clicks on specific regions.
* **Environment:** Pure Client-Side Web Application. Game state and saves are managed entirely via browser `localStorage` (or `IndexedDB` for larger saves), with no backend server.
* **Data:** GeoData (GeoJSON) for the 26 region borders, JSON for infrastructure and cities. These are static assets fetched via standard `fetch()` API. Mercator projection required (converting `lat/lon` into `x/y` coordinates on the Canvas).

## 2. CORE GAME LOOP
The game is divided into two strict phases:

### Phase 1: DAY (Strategy & Planning)
* **State:** Time is paused. No enemy attack animations occur.
* **Player Actions:** * Allocate budget for infrastructure repairs.
    * Relocate mobile Air Defense (AD) units.
    * Open "Points of Invincibility" (social shelters) and build protective structures (concrete covers/underground facilities).
    * Configure power grid routing (managing power flows between regions).
    * Implement rolling blackouts to preserve grid stability and prevent overload.
* **Controls:** Clicking on regions, opening information panels, adjusting priorities. State changes are saved to `localStorage`.

### Phase 2: NIGHT (Simulation & Consequences)
* **State:** Management UI is locked. The in-game night timer runs.
* **Actions:**
    * Visualize enemy attacks (missiles and drones represented as vectors flying toward targets).
    * Automatic engagement by player's Air Defense (interceptions).
    * Calculate hits: If an object is destroyed, its state changes, and connected grid lines are severed.
    * Visualize blackouts (regions go dark on the map in real-time).
    * Simulate random events: Grid overloads, accidents, and fires.
* **Transition:** Morning generates a daily report (damages, casualties, shifts in International Support) and auto-saves the game.

## 3. BUSINESS LOGIC & MATH

### 3.1. Resources & Metrics
* **Money (₴):** Global and regional budgets. Generated by population taxes.
* **International Support (%):** Slowly decays over time. Spikes if there are civilian casualties (bombings of residential areas). High support unlocks advanced Western AD systems and financial grants.
* **Depression (0-100%):** The primary loss condition metric. 
    * *Increases from:* Lack of electricity, lack of heat (in winter), bombings, and casualties.
    * *Decreases from:* Social projects, stable power.
    * *Formula Base:* `Base + (Bombing Impact) + (NoPower/NoHeat Penalty) - (SocialProjects Bonus)`.

### 3.2. Population (Pool of 10,000 Units)
10,000 abstract units are distributed across the 26 regions. Divided into 3 classes:
* **Autonomous (10%):** Have personal generators/resources. If local depression > 60%, they migrate abroad (permanent loss of tax revenue).
* **Mobile (50%):** The core tax base. They migrate to neighboring "green" (safe/powered) regions when depression gets too high, causing grid strain in the receiving region.
* **Rooted (40%):** Highly vulnerable. Cannot migrate and depend entirely on state infrastructure and social projects.

### 3.3. Power Grid & Objects
Infrastructure is parsed from static `infrastructure.json` with exact coordinates:
* **Global Generation (Tier 1/2):** NPPs (Nuclear - baseload), HPPs/PSPPs (Hydro - peak balancing), TPPs (Thermal - balancing).
* **Local Generation (CHPs/Factories):** Have a `heat: true` property. If these stop in winter, the region's depression multiplier increases drastically due to freezing.
* **Substations (750kV / 330kV):** Nodes in the energy graph. If a substation is destroyed, the power line breaks, isolating regions from the main grid.

### 3.4. Enemy AI
Scripted AI with escalating phases:
1.  Mass chaotic strikes on cities (Initial Shock).
2.  Targeted strikes on main substations and generation facilities (Systemic Blackouts).
3.  Pinpoint drone swarms to exhaust AD interceptor stocks before launching ballistics.

## 4. DATA MODELS
The system expects the following static input files in an `/assets` folder:
1.  `ukraine_regions.geojson`: Vector boundaries of the regions (for custom projection).
2.  `cities.json`: `[ { id, name, lat, lon, pop, tier } ]` - Used to render glowing city lights.
3.  `infrastructure.json`: `[ { id, name, category, type, lat, lon, capacity, status, heat, voltage } ]`.
4.  `rivers.json`: River geometry for map background aesthetics.
